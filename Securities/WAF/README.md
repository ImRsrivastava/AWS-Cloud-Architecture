# What and why is AWS WAF ?
    WAF can be a software, can be a appliances, can be service or can be a device that filters, monitors and blocks you http traffic to your web application or a web services.
    WAF protects your application from the common attacks such as SQL injections, cross-site scription, DDOS and the bot attacks, which can be at any layer of your application such application layer, libraries layer, middleware layer, os layer, data layer etc.

# What is the Features of AWS WAF ?

    2.  No operational overhead: Fully managed with ready to use built in Rules/Rules-Group.
    3.  Default and Customizable security: Customer customizable rules available.
    4.  Frictionless setup: No need to change application code.

# Component of AWS WAF ?
    Once you configure AWS WAF, First component need to create, that is Web ACL and need to define the protection strategy by adding Rules/Rules-Group. Rules can have a statement or condition which will define the inspection criteria and there will be an action define which would be taken if a web request matches the condition or the criteria in the statement.
